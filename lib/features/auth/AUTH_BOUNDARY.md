# Auth Boundary Note

## Контекст
- В auth-фиче используется сущность `RegistrationInput`.
- Она описывает данные auth-потока (sign up / sign in / текущая auth-сессия), а не профиль пользователя приложения.

## Решение
- `RegistrationInput` остается в `features/auth/domain/entities` как auth-ориентированная сущность.
- Полноценная пользовательская сущность будет введена отдельно в `user_profile` фиче.
- Auth не владеет данными профиля и не хранит профильную бизнес-логику.

## Правило границы
- Auth отвечает только за аутентификацию и состояние доступа.
- `user_profile` отвечает за профиль и его изменения.
- Преобразование auth-данных в профильные данные выполняется явным mapper/adapter в точке интеграции фич.

## Почему так
- Уменьшается связность между фичами.
- Избегается преждевременное смешивание auth и profile доменов.
- Проще развивать `user_profile` независимо от auth.
