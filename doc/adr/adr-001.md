# ADR 1: Файловая иерархия проекта

### Status: Предложение

---

### Date: 1.04.26

---

## Context:

Проект в формате интенсива, время ограничено.
Текущая цель: стабилизировать основу для 2 ключевых фич и задать формат для следующих.
Требование: структура должна поддерживать умеренный рост без радикального пересмотра в ближайшие недели.
Основной прогнозируемый риск: разрастание конфигурации зависимостей и потеря читаемости структуры.

## Decision:

Принять feature-first организацию: каждая бизнес-фича хранится в своем модуле.
Внутри фичи использовать единый шаблон слоев: `data / domain / presentation`.
Общие инфраструктурные элементы хранить отдельно от фич (`core, config`).
Новые фичи добавлять только по этому шаблону; отклонения фиксировать отдельным ADR.
В этом ADR не фиксируются: стратегия навигации, детализация DI, UI-дизайн-система.

## Consequences:

- Positive:
  Ускоряется добавление новых фич за счет повторяемого шаблона.
  Снижается риск хаотичного размещения кода между слоями и модулями.

- Negative:
  Появляется риск избыточной вложенности для небольших фич.
  Требуется дисциплина команды: одинаковый нейминг и следование шаблону.
